1. 🔄 A Git & GitHub Munkamenet (Branching & Remotes)

A független és tiszta fejlesztés érdekében használjátok a Feature Branch modellt:

  • Saját fejlesztési ágak (Branch-ek) használata:
    Ne dolgozzatok közvetlenül a main ágon. Ha egy új funkción (pl. hangüzenetek vagy MSN Custom Status) dolgozol, kérd meg az Antigravity-t:
    "Hozz létre egy feature/voice-messages git ágat és ott valósítsuk meg a funkciót."
  • Pull Request (PR) küldése Alexnak:
    Amikor elkészül egy funkció, az Antigravity segítségével vagy a GitHubon küldj egy Pull Request-et Alex repójába (adamcsaka/ChatTT). Alex átnézheti és egy kattintással beolvaszthatja a fő repóba.
  • Upstream frissítések lekérése 1 paranccsal:
    Bármikor, amikor Alex fejleszt valamit a saját repójában, csak mondd ezt az Antigravity-nek:
    "Frissítsük és mergeljük a legújabb upstream módosításokat Alex repójából!"
    Az Antigravity lefuttatja a git fetch upstream és git merge upstream/main parancsokat, és kezeli az esetleges ütközéseket.

2. 🤖 Az Antigravity AI Okos Használata

Az Antigravity nem csak kódot ír, hanem automatizálja az egész munkafolyamatot:

  • Automatikus Élő Tesztelés és Weboldal Telepítés (GitHub Pages):
    Minden alkalommal, amikor elkészül egy frissítés, az Antigravity automatikusan pushol a GitHubra. Az élő tesztkörnyezeted mindig elérhető a https://sid14925.github.io/ChatTT/ címen.
  • Automatikus EXE Fordítás:
    Nem kell manuálisan fordítanotok a Windows futtatható fájlokat; az Antigravity a háttérben legenerálja a legfrissebb .exe fájlt és a hordozható ZIP-et.
  • Összetett Funkciók Tervezése (Planning Mode):
    Ha egy komolyabb újítást szeretnétek (pl. Videóhívás WebRTC-vel), használd a /grill-me parancsot, vagy kérd meg az Antigravity-t, hogy készítsen egy részletes implementation_plan.md tervet a megvalósítás előtt.
